C川瀬さんに渡す前チェックリスト（青山さん）

  1. AI アシスタントのアカウント招待メールを送付 ― Team プランのメンバー招待、または川瀬さん名義の個人プランを用意（アカウント共有は NG：利用規約違反＆停止リスク）
  2. GitHub Collaborator 招待 ― gift-org/form-sales に招待。招待前に川瀬さんが clone すると "Repository not found" で詰まるため、招待メール受信を先に確認 してもらう
  3. 連絡先の電話番号を Chatwork DM で共有 ― 資料には電話番号を書かない。Chatwork DM 経由で「試運転期間中の緊急連絡先」を送っておく
  4. 送信元情報は川瀬さんが AI に「セットアップして」と言うだけで入る ― AI が会話で会社名・担当者名・カナ・メール・電話・URL・初期文面を聞き出し、DB (sender_identity / templates) に入れて sandbox プリフライトまで自動で通す。後から変えたい場合のみ管理画面 /sender ／ /templates を使う。.env は触らない。青山さんは値を見ない。
    共有が必要な秘密（DB パスワード等）がある場合は 1Password / Bitwarden 等のパスワードマネージャの共有ボールト を使う（メール本文・パスワード付き zip は NG）
  5. 実在先リストは青山さんが 1 人で切り出し ― 「担当分けメモ」の割当を青山さんが決めて、川瀬さん側に渡す分だけを CSV 化。両 Mac 間で同一 domain が重複しないよう青山さんが最終責任を持つ
  6. 全停止ボタンの説明を Chatwork DM か初回対面で 1 回 ― 「変だと思ったら押して OK。復旧は AI に頼めば戻せる」
  7. 撤退ライン（Go / Pivot / Kill）を Chatwork DM で 1 回 ― docs/12-pre-poc.md の 3 判定を要点だけ。ハンズオン §7 にも記載済み
  8. 初回だけ 15〜30 分横で見る ― セットアップ〜1 件目の確認待ち承認まで、隣で見守る（アカウント作成の初回だけ）

D試運転を始められる前提

  1. 法務 OK ― プライバシーポリシーの弁護士チェック済（2026-06-30 取得）
  2. 青山さん Mac が起動 ― 送信元情報を /sender または AI「セットアップして」で登録済み
  3. 川瀬さん Mac にセットアップ完了 ― AI アシスタント初期化と gh 認証まで終わり、AI に「セットアップして」で自社情報＋初期文面が DB に入っている（.env の SENDER_* 手書きは不要）
  4. 実在先リストの担当分けが済んでいる ― 「担当分けメモ」に沿ってスプレッドシート更新済み

E青山さんの承認導線（試運転由来 Issue / PR）

川瀬さん由来の Issue / PR は trial-feedback ラベル で自動分類されます。青山さんは 1 日 1 回下の 2 本を見るだけで OK。

  • 試運転由来 Issue 一覧: issues?label=trial-feedback
  • 試運転由来 PR 一覧: pulls?label=trial-feedback

※ ラベル trial-feedback と urgent は事前にリポジトリに作成しておく（gh label create trial-feedback ／ gh label create urgent）。
