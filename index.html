川瀬さん用 ハンズオン
― Mac で「AI に頼むだけ」で試運転を始める

川瀬さんの Mac に AI アシスタントを入れて、あとは日本語で頼むだけで フォーム営業の試運転が動くようになります。所要 30 分。 困ったら青山さんに聞いてください。

1はじめに

この資料は、川瀬さんの Mac 1 台で GIFT フォーム営業を試運転するための手順です。

  • 入れるのは AI アシスタント（＝AI が Mac の中で作業してくれる相棒）
  • 川瀬さんが自分でやるのは ターミナルを開く／コピペして Enter／パスワード入力／ブラウザで承認ボタンを押す の 4 つだけ
  • それ以外は AI アシスタントが全部やります。分からないコマンドは覚えなくて OK
  • 詰まったら「これで詰まった」と AI に言ってください。読んで直してくれます

2Mac に AI アシスタントを入れる（初回だけ 15 分）

2-1. ターミナルを開く

Mac の画面右上の 虫めがねアイコン（Spotlight） を押して、ターミナル と打って Enter。
黒か白の四角い画面が出てくればそれです。以下のコマンドは、そこにコピペして Enter で実行します。

2-2. Homebrew を入れる（Mac の道具箱）

下の 1 行をコピペして Enter。途中で Mac のログインパスワードを聞かれます（画面に文字は出ませんが打てています）。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

最後に「Next steps:」と出たら、その下に書いてある eval "$(...)" の行もそのままコピペして Enter してください（PATH を通す作業です）。

2-3. Node と Git を入れる

brew install node git

2-4. AI アシスタントを入れる

npm install -g @anthropic-ai/claude-code

これで claude というコマンドが使えるようになります。

2-5. 起動する

claude
初回だけ青山さんに手伝ってもらう

初回起動時にログインを聞かれます。青山さんに電話して、アカウント共有 or 一時トークンを発行してもらってください。ブラウザが自動で開きます。以降は聞かれません。

3試運転するリポジトリ（プログラム一式）を取り込む

下を 1 行ずつコピペして実行してください。

cd ~/Documents git clone https://github.com/gift-org/form-sales.git cd form-sales claude

最後の claude でこのプロジェクトの中で AI アシスタントが立ち上がります。ここから先は日本語で頼むだけ。

GitHub への認証が要ったら

「認証してほしい」と AI に言えば gh auth login を案内してくれます。川瀬さんは表示された URL をブラウザで開いて Authorize のボタンを押すだけ。

5バグや違和感を見つけたら（重要）

「気になった」段階で言葉にして AI に頼んでください。
AI がその場で GitHub に Issue（＝報告カード）を切って、青山さんが見に行けるようにします。
修正案まで明確なら、AI がそのまま 修正提案（PR） まで作ります。

AI が自動でやること

1

現象を要約

川瀬さんが話した内容を「何をしたら」「何が起きた」「期待していた挙動」に整理します。

2

GitHub Issue を立てる

gh issue create で自動起票。ラベルは trial-feedback。青山さんが後で気付けます。

3

影響が小さければ PR まで作る

修正案が明確で 1〜2 ファイルの変更で済む場合、feature ブランチで PR を作ります。青山さんがマージ判断。

権限が足りないとき

gift-org のメンバー招待がまだの場合、Issue だけ 立てて「青山さんへ引き継ぎ」と本文に書きます。青山さんが後で対応。
川瀬さんは何もしなくて OK。

青山さんが承認する導線

Issue → github.com/gift-org/form-sales/issues
PR → github.com/gift-org/form-sales/pulls
青山さんは 1 日 1 回、上を軽くチェックして、対応方針を決める or マージします。

6画面の使い方（5 つ + 停止ボタン）

ブラウザで http://127.0.0.1:3000/ を開いて、上のメニューから移動します。
分からなくなったら AI に「◯◯の画面を開いて」と頼んで OK。

画面 何が見える／何をする
成績
（ダッシュボード）		 今日／今週の送信数・返信数・アポ数。まずここで全体を見る。
確認待ち
（人間キュー）		 実在先に送る前の最終チェック。「送信 / 補完して送信 / スキップ」 の 3 択。迷ったらスキップ。
文面
（テンプレ管理）		 A/B/C/D の 4 種類。反応率が併記される。直したい文面を選んで書き換え。
除外先
（サプレッション）		 もう送らない会社・ドメインの一覧。「営業お断り」返信が来た会社はここに追加。
設定 1 日の送信上限、送信時間帯、送信間隔。試運転中は既定値のままで OK。
画面右上の 赤い「全停止」ボタン

押すと 送信が即座に止まります（緊急停止）。
変だと思ったら押して大丈夫。復旧は AI に「送信を再開して」で戻せます。

7困ったとき（詰まり事典）

まず、下の言い方で AI に投げてください。8 割はこれで解決します。

エラーが出た。読んで対処して

それでもダメなら青山さんに電話。

PostgreSQL に接続できません

AI に「Postgres を起動して」と頼めば brew services start postgresql@16 を実行してくれます。

画面が真っ白／管理画面が開かない

AI に「管理画面のログを見て原因を教えて」と頼む。多くはポート衝突か DB 未起動。

「認証してください」と GitHub に言われる

AI に「gh auth login して」と頼む。ブラウザで開いた URL の Authorize を押すだけ。

「.env の値が足りません」と止まる

AI に「.env の必須項目を教えて。埋め方も出して」で誘導してくれます。

送信が止まらない気がする

まず管理画面右上の 赤い「全停止」ボタン を押す。落ち着いてから AI に「状況を教えて」。

何をしてよいか分からない

触らない。青山さんに電話。壊れないので大丈夫。
